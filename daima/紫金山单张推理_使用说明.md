## ⚙️ 技术细节

### 模型加载

Cell会从全局变量 `models_dict` 中获取已加载的模型：

```python
if 'models_dict' in globals() and 'LoveDA' in models_dict:
    model = models_dict['LoveDA'].get(m_name)
```

**必须条件**：前面Cell已执行，`models_dict` 已初始化

### 图像处理流程

1. **读取**: PIL Image 读取PNG图像
2. **预处理**: 
   - 缩放到模型输入大小 (512 或 1024)
   - ImageNet标准化 (mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225])
3. **推理**: 模型前向传播，argmax后得到类别标签
4. **后处理**: 统一缩放到 512×512，转换为 uint8 [0-255]
5. **保存**: PIL Image 保存为PNG

### 模型输入大小

- **UNet, DeepLabV3+**: 512×512 (标准CNN)
- **SAM2_Tiny, MobileSAM, SAM2.1_Tiny, MobileSAMV2**: 1024×1024 (Vision Transformer)

**注意**: 所有预测都会缩放到 512×512 统一输出

## 🐛 常见问题

### Q1: 报错 "KeyError: 'LoveDA'"

**原因**: 前面Cell尚未运行，`models_dict` 未初始化  
**解决**: 先运行 Cell 1-9 加载模型

### Q2: 报错 "ZJS_DATASET_DIR not found"

**原因**: 紫金山数据集不在指定位置  
**解决**: 修改路径为实际位置
```python
ZJS_DATASET_DIR = "/your/path/to/2024-seg"
```

### Q3: 报错 "CUDA out of memory"

**原因**: GPU内存不足  
**解决**: 降低批处理大小或使用CPU
```python
DEVICE = "cpu"  # 强制使用CPU
```

### Q4: 推理很慢？

**原因**: SAM模型 (特别是1024×1024输入) 推理较耗时  
**预期耗时**:
- UNet/DeepLabV3+: 各 ~300ms
- SAM2/MobileSAM: 各 ~1-2秒
- **总计**: ~6-10秒 (首次可能久一点)
